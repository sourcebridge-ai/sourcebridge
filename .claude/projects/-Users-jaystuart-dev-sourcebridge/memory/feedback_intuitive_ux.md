Every feature must be super intuitive and easy to use. UX is not an afterthought.

**Why:** Jay has consistently emphasized this across features — the comprehension settings page, the understanding score, the reports wizard. If it isn't intuitive, it won't be used (matches the plan's UI/UX principle #8).

**How to apply:** Before building any UI surface, think about the operator who has never seen this screen before. Can they accomplish their goal in under 60 seconds without reading docs? Apply progressive disclosure — simple mode by default, advanced behind a toggle. Empty states should teach. Errors should explain and offer one-click fixes. Labels should be human, not technical jargon.
